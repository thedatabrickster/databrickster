# Agentic Coding

:::note In progress
The **[VS Code](/agentic-coding/vscode/intro)** subtopic is now built out in full — a
nine-lesson track, taught the same way as [Databricks AI](/docs/intro). **Claude Code**
is next on the roadmap.
:::

> Notebooks are great for exploring. But real AI projects are *software* — with repos,
> tests, reviews, and CI. **Agentic coding** is building that software faster by
> pairing with AI coding agents in a real development environment.

## What this topic will cover

This track is tool-focused and useful whether you build on Databricks or anywhere
else. It's organized into two subtopics:

- **[VS Code](/agentic-coding/vscode/intro)** ✅ *available now* — setting up a professional editor for Databricks and AI work: the Databricks extension, Databricks Connect, Asset Bundles, a repo-first workflow, local debugging & testing, and AI assistants over MCP.
- **[Claude Code](/agentic-coding/claude-code/intro)** — using an agentic coding assistant to scaffold agents, write tests, refactor, debug, and drive multi-step changes, with a human in the loop.

Cross-cutting lessons will also cover:

- **Why the IDE, not just notebooks** — when to graduate from notebooks to a versioned, testable codebase (and how they work together).
- **A repo-first agent workflow** — structuring a Databricks AI project as code with CI/CD.
- **Testing & debugging agents locally** — tracing, evaluation, and reproducing failures outside a notebook.
- **MCP in your editor** — connecting your coding agent to Databricks tools (Unity Catalog functions, Vector Search, Genie) via the Model Context Protocol.

## How it connects to the rest of the site

The [Databricks AI](/docs/intro) track teaches *what* to build; this track teaches
*how to build it fast and professionally*. It also pairs naturally with
[Part 11 · Beyond Databricks](/docs/beyond-databricks/concepts-are-portable), since a
good editor + agent workflow is portable across any AI stack.

*Two tools, one goal: ship AI faster. On the roadmap.*

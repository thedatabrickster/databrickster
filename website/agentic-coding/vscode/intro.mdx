> A great editor turns notebook experiments into shippable software. This subtopic is
> about making **VS Code** a first-class home for Databricks and AI work.

## Planned lessons

- **Set up the Databricks extension for VS Code** — connect to your workspace, browse Unity Catalog, and run code on remote compute.
- **Databricks Connect** — run Spark and Databricks code from your local editor against a remote cluster.
- **Asset Bundles from the editor** — define jobs, agents, and endpoints as code and deploy them without leaving VS Code.
- **A repo-first workflow** — structure, lint, test, and version an AI project like real software.
- **Debugging & tests** — step through agent logic locally, write unit/eval tests, and reproduce failures outside a notebook.
- **Extensions that help AI engineers** — Python, Jupyter, linters, and the ones worth installing.

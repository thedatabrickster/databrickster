# DAIS 2026 — The Modern Databricks Stack

A fast, opinionated tour of what Databricks announced at **Data + AI Summit 2026**
(June 16–17, 2026). The keynote theme was the **"Modern Databricks Stack"**: an open
lakehouse foundation, unified governance, and — the big shift — an **agentic** layer for
work and development sitting on top.

Use the **left pane** to jump to any update. Each page is short and consistent:

- **Overview** — what was announced.
- **Where it fits** — the use cases and the layer of the stack it lands in.
- **Where it fails** — limits, gaps, and hype-vs-reality.
- **Response** — the practical move (adopt / pilot / watch / ignore) and market context.
- **Personal impression** — a candid verdict.

## The stack, top to bottom

```text
Agentic Work & Development   Agent Bricks · Genie (Code/One/Agents/ZeroOps/Ontology) ·
                             Omnigent · AI Platform (Real-Time ML) · Governed Vibe Coding
        ↑
Unified Governance & Security   Unity Catalog · Unity AI Gateway · OpenSharing ·
                                Platform Security & Compliance
        ↑
Open Lakehouse Foundation    Lakebase + LTAP · Lakehouse//RT · Lakeflow ·
                             AI-First Data Engineering · Delta / Iceberg (open formats)
        ↑
Runs on                      AWS · Azure · GCP (cloud substrate)
```

*The narrative: the lakehouse foundation is now real-time and transactional; governance
extends to models, agents, and MCP; and an agentic layer automates the "other 99%" of
data + AI work.*

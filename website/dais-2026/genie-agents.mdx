# Genie Agents

> Genie Spaces with hands: the same curated, benchmarked context, now allowed to act across your business tools instead of just answering.

*Announced at Data + AI Summit 2026 (June 16–17, 2026). Source: [Databricks blog](https://www.databricks.com/blog/introducing-genie-one-genie-ontology-and-genie-agents).*

**Stage:** <StatusBadge stage="Not stated" /> — the announcement did not give a release stage.

## What it is

- **Genie Spaces is becoming Genie Agents** — curated, domain-specific agents built on the more than one million Genie Spaces customers have already created.
- **They take autonomous action** using the Genie One toolset: MCP connections, scheduled tasks, document and artifact generation, and writes to external systems — multi-step workflows, not just answers.
- **They reason over unstructured data**, grounding in documents, files, and knowledge sources alongside governed tables and views.
- **You spin one up from a single prompt** in Genie One or Genie Code, then scope it, benchmark it, and share it for teammates to use or fork.
- **It inherits Genie lineage** — the same verified logic, scoping, and benchmarks that made Genie Spaces trustworthy carry forward into the agents.

## Where it fits

- **Semantic / consumption layer** of the Modern Databricks Stack — self-serve analytics *and* action on top of Unity Catalog-governed data. See [the overview](/dais-2026).
- Lets **domain experts scale themselves** — trusted rules, data, and workflows become an agent the whole team can lean on instead of a person answering the same questions.
- Pairs with **Unity Catalog**, so agent reads and writes inherit existing permissions, lineage, and governance rather than routing around them.
- Sits next to **Genie One** (the coworker in Slack, Teams, and mobile) and **Genie Ontology** (the automatic context graph), which is where the grounding comes from.
- The real shift is **Q&A to action** — MCP tool use and scheduled writes turn a good answer into a completed task, which is genuinely new territory for a BI-adjacent tool.

## Where it falls short

- **Curation is load-bearing, and most orgs skip it.** Benchmarks and scoping are only as good as the trusted rules behind them; point one at a raw, undocumented schema and it inherits every ambiguity in the model. This is not a text-to-SQL you turn on — it's one you invest in.
- **Text-to-SQL plus autonomy multiplies the blast radius.** A wrong assumption used to produce a wrong chart someone could sanity-check. An agent with write access and no oversight turns that same assumption into a real change in a downstream system.
- **Governance of generated SQL and actions is the open question.** Unity Catalog governs *access*, but who reviews the SQL the agent writes, approves an external-system write, or audits a scheduled task before it runs? That review layer is where this succeeds or fails.
- **The Genie naming is a mess.** Genie Agents, Genie Spaces, Genie One, Genie Code, Genie Ontology — expect real confusion about what is a space, an agent, and a coworker, both internally and in the docs.
- **Curation doesn't port.** Trusted rules, benchmarks, and MCP wiring are Databricks-specific work. Versus Cortex Analyst or ThoughtSpot the differentiator is native Unity Catalog governance and action across tools, not the NL-to-SQL itself — which is table stakes now.

## My take

**Pilot, don't roll out.** The action-taking value is real, but it only shows up after you've done the scoping and benchmarking work — so pilot on one well-modeled domain where you already trust the tables, not across the warehouse. If you already run curated, benchmarked Genie Spaces on clean Unity Catalog data, the migration path is natural and you should start now.

The thing to get right on day one is autonomy. Gate external-system writes and scheduled actions behind human review until an agent's benchmarks have earned trust, and keep someone who actually understands the data in the loop before it acts on a decision. The single-prompt creation flow will demo beautifully; the durable question is whether scoping holds up on messy enterprise data and documents — and that's on you, not the model.

Autonomous business-tool agents were the most over-promised category of the summit, and every vendor is racing here. Databricks' honest edge is that the governance and the data live in the same place — that's worth more than another text-to-SQL box.

**Bottom line:** Pilot on one curated, benchmarked domain, keep external writes on a short leash, and don't let an agent act until a human who knows the data has vouched for its benchmarks.

# Databricks Features — Architecture Guides

Executive-style, decision-focused reference for the Databricks platform. Each feature has
a short guide (3–5 minute read) written for **CIOs, CTOs, VPs of Engineering, Directors of
Data, and Enterprise / Solution / Platform Architects** — architecture placement, fit,
dependencies, alternatives, cost, and a decision matrix. No tutorials, no marketing.

:::note[Coverage]
This is a **living catalog** — the goal is to cover *every* Databricks feature,
alphabetically. Linked entries have a published guide; the rest are queued and being added
in batches. If a feature you need isn't linked yet, ask and it'll be prioritized.
:::

## How each guide is structured

Every feature guide follows the same 19-section template so they're directly comparable:

- Executive Summary · Primary Purpose · Where It Fits in the Lakehouse · Architecture Placement
- Ideal Use Cases · Poor Fit · Best Suited For · Enterprise Value
- Dependencies · Alternatives · Strengths · Limitations · Common Mistakes
- Design Recommendations · Decision Matrix · Quick Comparison · Maturity · Cost · Executive Takeaway

## The Lakehouse placement model

Where features sit, top to bottom:

```text
Users / Executives / Business
        ↓
User Interfaces        (AI/BI Dashboards, Genie, Databricks Apps, Notebooks, SQL Editor)
        ↓
Developer Experience   (Assistant, Repos/Git, Asset Bundles, CLI/SDK, Connect)
        ↓
AI & Machine Learning  (Mosaic AI: Vector Search, Model Serving, Agent Framework, MLflow)
        ↓
Data Engineering       (Lakeflow: Connect, Declarative Pipelines, Jobs; Auto Loader, Spark)
        ↓
Analytics / Query      (Databricks SQL, SQL Warehouses, Photon, Materialized Views)
        ↓
Governance             (Unity Catalog, Lineage, ABAC, Audit Logs, Lakehouse Monitoring)
        ↓
Storage                (Delta Lake, Volumes, Delta Sharing, DBFS)
        ↓
Compute                (Serverless, Clusters, SQL Warehouses, Pools, Photon, DBR)
```
